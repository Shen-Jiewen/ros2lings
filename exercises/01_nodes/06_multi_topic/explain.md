# 多话题通信 — 一个节点管理多个话题

## 概念

在实际机器人系统中，一个节点通常需要处理多个话题。
例如，一个导航节点可能同时订阅激光雷达和里程计话题，
同时发布速度命令和路径规划话题。

## 架构示意

```
              MultiTopicNode
          ┌───────┴───────┐
          │               │
   ┌──────┴──────┐ ┌──────┴──────┐
   │  Publisher   │ │  Publisher   │
   │  "status"   │ │  "command"   │
   └──────┬──────┘ └──────┬──────┘
          │               │
   ┌──────┴──────┐ ┌──────┴──────┐
   │ Subscriber  │ │ Subscriber  │
   │  "status"   │ │  "command"  │
   └─────────────┘ └─────────────┘
```

## 关键点

- 一个节点可以创建任意数量的 Publisher 和 Subscription
- 每个话题有独立的消息队列
- 不同的话题可以使用不同的消息类型
- 每个订阅者有自己的回调函数
- 话题命名应有意义，便于系统调试和理解
